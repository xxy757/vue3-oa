# AGENTS.md - OA SaaS 项目开发规范

## 前端设计规范

### 整体风格定位

- **参考基准**：Ant Design Pro 后台管理系统、Google Material Design、Apple Human Interface
- **核心原则**：简洁大气、留白充足、层次分明、信息密度适中
- **视觉特征**：大面积留白、轻阴影、圆角卡片、克制使用颜色

---

### 色彩体系

#### 主色（Primary）

- 主色：`#1677FF`（Ant Design 蓝）
- 悬停：`#4096FF`
- 点击：`#0958D9`
- 浅底：`#E6F4FF`（用于背景高亮、选中态）
- 仅用于：主要操作按钮、链接、选中态、关键数据强调

#### 中性色（Neutral）

- 标题文字：`rgba(0, 0, 0, 0.88)`
- 正文文字：`rgba(0, 0, 0, 0.65)`
- 辅助文字：`rgba(0, 0, 0, 0.45)`
- 禁用文字：`rgba(0, 0, 0, 0.25)`
- 边框：`#D9D9D9`
- 分割线：`#F0F0F0`
- 页面背景：`#F5F5F5`
- 卡片背景：`#FFFFFF`

#### 语义色

- 成功：`#52C41A`
- 警告：`#FAAD14`
- 错误：`#FF4D4F`

### 组件风格规范

#### 按钮

- 主按钮：实心主色背景 `#1677FF`，白色文字
- 次要按钮：白底灰边框，hover 时边框变主色
- 文字按钮：无边框无背景，主色文字
- 危险按钮：红色系 `#FF4D4F`
- 尺寸：默认 `middle(32px)`，小按钮 `small(24px)`
- 不使用圆角药丸形按钮

#### 表格（DataTable）

- 表头背景：`#FAFAFA`
- 斑马纹：不使用（保持简洁）
- 行悬停：`#FAFAFA`
- 边框：仅底部边框 `1px solid #F0F0F0`
- 空状态：居中图标 + 描述文字
- 单行高度：`54px`
- 操作列：文字按钮，主要操作用 primary 色，危险操作用 error 色

#### 表单

- 标签位置：`label-placement="left"`，标签宽度 `80px`
- 输入框高度：`32px`
- 验证错误：红色文字提示在输入框下方
- 弹窗表单：使用 `n-modal` preset="dialog"

#### 卡片（Card）

- 白色背景 + 轻阴影
- 标题区：16px 中等字重，右侧可放操作按钮
- 内容区：20px 内边距
- 不使用多余装饰线

#### 弹窗（Modal/Dialog）

- 宽度：表单类 `520px`，详情类 `640px`
- 圆角 `8px`
- 顶部标题栏 + 底部按钮栏

#### 标签（Tag）

- 尺寸：`small`
- 状态映射：成功绿、警告橙、错误红、进行中蓝、默认灰
- 不使用彩色背景标签（除非是状态标识）

#### 分页（Pagination）

- 右对齐
- 显示总数 `show-total`
- 显示页码选择器 `show-size-picker`

#### 统计卡片（Statistic）

- 上方：12px 辅助色标签
- 中间：30px 大号数字
- 下方：12px 辅助说明（同比/环比数据）
- 左侧可配图标

### 图标规范

- 使用 `@vicons/ionicons5` 图标库
- 图标尺寸：默认 `18px`，按钮内 `16px`
- 图标颜色跟随文字颜色，不单独设色
- 操作按钮需要配图标：新增 `AddOutline`、编辑 `CreateOutline`、删除 `TrashOutline`、搜索 `SearchOutline`

---

### 动效规范

- 过渡时间：`200ms` - `300ms`
- 缓动函数：`ease-in-out`
- 使用场景：hover 背景色变化、弹窗出现/消失、展开/折叠
- 不使用：弹跳、旋转、闪烁等花哨动效

---

### 不允许的做法

- 不要使用渐变背景（按钮、卡片、页面都不用）
- 不要使用药丸形圆角按钮
- 不要在卡片上使用彩色边框装饰
- 不要使用emoji作为图标
- 不要过度使用彩色标签，保持克制
- 不要使用过粗的分割线（不超过 1px）
- 不要使用纯黑 `#000` 作为文字色
- 表格不要同时使用斑马纹和悬停高亮

## 代码规范

### 组件导入

- 从 `naive-ui` 按需导入组件，不要用全局注册
- 图标从 `@vicons/ionicons5` 导入

### 样式

- 使用 `<style lang="scss" scoped>`
- 通过 CSS 变量引用色彩：`var(--primary-color)` 等
- 响应式断点：`768px` / `1024px` / `1280px`

### TypeScript

- 不使用 `any` 类型（除非类型确实无法确定，需加注释说明）
- 接口定义放在 `types/` 目录
- 组件 props 使用 `defineProps<T>()` 泛型语法

### 不写注释

- 代码不添加任何注释，除非用户明确要求

## 常见错误防范

### 禁止重复导出同名变量

- 同一个文件中 **不得** 出现两个 `export const` / `export function` 使用相同的名称
- 生成或编辑路由、常量等文件时，必须先检查文件中是否已存在同名导出
- 如果需要替换，应先删除旧定义再写新定义，而不是在文件末尾追加

### 禁止重复导入 SCSS 变量文件

- `vite.config.ts` 已通过 `css.preprocessorOptions.scss.additionalData` 全局注入 `@import "@/styles/variables.scss"`
- 所有 `.scss` 文件中 **不得** 再次 `@import` 或 `@use` `variables.scss`，否则会导致变量重复定义和 Sass 弃用警告
- `src/styles/index.scss` 等入口样式文件只需导入其他样式文件（如 `reset.scss`），无需导入 `variables.scss`

## 调试与修复流程规范

### 先全面诊断，再动手修复

- 修复 Bug 时 **必须先通盘扫描**，列出所有问题后再逐一修复，禁止看到表面报错就逐个修
- 排查范围至少包括：
  - TypeScript 类型检查（`vue-tsc --noEmit`）
  - 所有 `.ts` 文件的语法完整性（合并行、缺失导入、多余 token）
  - 构建配置完整性（`vite.config.ts` 中的插件注册、proxy 配置）
  - Mock 数据与 Store 请求路径是否匹配（URL 前缀、单复数）
  - Mock 返回数据结构是否与 TypeScript 类型定义一致
- 典型反面案例：只修了路由重复导出，却遗漏了 `constantRoutes` 缺失、`request.ts` 语法损坏、mock 未注册、mock URL 不匹配、mock 返回数据缺字段等一系列问题，导致反复修复多次
